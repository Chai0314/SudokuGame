# 数独游戏功能增强 - 产品需求文档

## Overview
- **Summary**: 为现有的数独游戏添加三个新功能：完成游戏时的提示框、填数值时的标记功能，以及在本地存储中保存最快完成时间信息。
- **Purpose**: 提升游戏体验，增加游戏的可玩性和竞争性，让用户能够追踪自己的最佳成绩。
- **Target Users**: 数独游戏爱好者，希望挑战自我并记录最佳成绩的玩家。

## Goals
- 实现游戏完成时的提示框，展示完成时间和最快记录
- 为填数值的单元格添加标记功能，允许用户标记可能的数字
- 在本地存储中保存不同难度级别的最快完成时间

## Non-Goals (Out of Scope)
- 不涉及游戏核心逻辑的修改
- 不添加新的游戏难度级别
- 不修改现有的UI布局和样式

## Background & Context
- 现有数独游戏已经实现了基本的游戏功能，包括生成谜题、验证答案、提示功能等
- 游戏使用 Vue 3 + TypeScript 开发，状态管理通过 composables 实现
- 目前游戏在完成时会显示简单的提示信息，但没有记录和展示最快完成时间
- 玩家在填写数字时没有标记功能，无法记录可能的候选数字

## Functional Requirements
- **FR-1**: 游戏完成提示框增强
  - 当玩家完成数独游戏时，显示一个提示框
  - 提示框应显示本次完成时间
  - 提示框应显示该难度级别的最快完成时间
  - 如果本次完成时间是新的最快记录，应给予特殊提示

- **FR-2**: 本地存储最快时间
  - 在本地存储中保存不同难度级别的最快完成时间
  - 游戏完成时，自动更新对应难度的最快时间（如果本次时间更快）
  - 游戏加载时，从本地存储中读取最快时间数据

- **FR-3**: 单元格标记功能
  - 允许用户为空白单元格添加标记（候选数字）
  - 标记应显示在单元格的角落，不影响主数字的显示
  - 支持添加和移除标记
  - 标记功能应与现有的数字输入功能共存

## Non-Functional Requirements
- **NFR-1**: 性能
  - 标记功能不应影响游戏的响应速度
  - 本地存储操作应在后台完成，不影响用户体验

- **NFR-2**: 可用性
  - 标记功能的操作应简单直观
  - 提示框的设计应与现有UI风格一致

- **NFR-3**: 兼容性
  - 本地存储功能应兼容主流浏览器
  - 标记功能应在不同屏幕尺寸上正常显示

## Constraints
- **Technical**: 基于现有的 Vue 3 + TypeScript 代码base，不引入新的依赖
- **Dependencies**: 依赖浏览器的 localStorage API 进行本地存储

## Assumptions
- 玩家使用支持 localStorage 的现代浏览器
- 玩家希望记录自己的最佳成绩并与之前的成绩进行比较
- 玩家在解决数独时需要标记功能来辅助思考

## Acceptance Criteria

### AC-1: 游戏完成提示框
- **Given**: 玩家完成数独游戏
- **When**: 游戏验证通过后
- **Then**: 显示提示框，包含本次完成时间和最快记录
- **Verification**: `human-judgment`

### AC-2: 本地存储最快时间
- **Given**: 玩家完成数独游戏
- **When**: 完成时间比存储的最快时间快
- **Then**: 本地存储中的最快时间被更新
- **Verification**: `programmatic`

### AC-3: 单元格标记功能
- **Given**: 玩家选择一个空白单元格
- **When**: 玩家使用特定操作添加标记
- **Then**: 标记显示在单元格中
- **Verification**: `human-judgment`

### AC-4: 标记显示效果
- **Given**: 单元格有标记
- **When**: 查看该单元格
- **Then**: 标记显示在单元格角落，不影响主数字显示
- **Verification**: `human-judgment`
